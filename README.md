<h1>S72_Mallikarjun_Capstone_Event_Hub</h1>

<h3>Week 1 - Planning & Initial Development:</h3>
Day 1: Finalize project scope, gather all requirements, and design the architecture.

Day 2: Create a database schema for categories, events, sub-events, and packages.

Day 3: Set up the project environment (frontend, backend, and database).

Day 4: Design the homepage layout with event thumbnails and images.

Day 5: Implement the homepage functionality, including redirections to the services page.

Day 6: Develop the services page with main event categories (e.g., Birthday, Marriage).

Day 7: Add sub-event components under each main event (e.g., decoration, cake).

<h3>Week 2 - Core Functionalities:</h3>
Day 8: Add package options (Gold, Platinum) for each event category.

Day 9: Implement the customization feature for users to select their preferences.
 
Day 10: Create a form for users to describe their dream events.

Day 11: Build a page to display event images, cakes, and global event locations.

Day 12: Implement login and sign-up functionality for user authentication.

Day 13: Create a user dashboard to showcase major events and dream ideas.

<h3>Week 3 - Advanced Features & Integration:</h3>
Day 14: Set up the contact page with details, Instagram ID, and footer information.

Day 15: Integrate the frontend with the backend and connect the database.

Day 16: Add dynamic content loading for event categories, packages, and images.

Day 17: Test all functionalities thoroughly to ensure everything works seamlessly.

<h3>Week 4 - Finishing Touches & Deployment:</h3>
Day 18: Optimize the platform for performance and fix any bugs identified during testing.

Day 19: Add final touches to the UI/UX design to make it visually appealing.

Day 20: Deploy the website on a hosting platform and prepare for the final demo .
